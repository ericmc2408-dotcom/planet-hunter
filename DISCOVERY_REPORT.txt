METHODOLOGY
═══════════════════════════════════════════════════════════════════════

1. Data Acquisition
   - Source: TESS public archive (MAST)
   - Target selection: Systematic sampling across TIC catalog
   - Light curves downloaded via Lightkurve API

2. Transit Detection
   - Algorithm: Box Least Squares periodogram
   - Period range: 0.5 - 20 days
   - Minimum transits required: 2

3. Candidate Validation
   - Outlier removal
   - Light curve flattening
   - Statistical significance filtering
   - Binary star rejection (depth > 10%)

4. Classification
   - Score = Power × Transit_Depth × 100
   - Threshold: Score > 50 and < 10,000
   - Manual validation: Light curve inspection
